# 繼續瀏覽使用不安全 SSL 憑證網站的方法

一直常常忘記 `thisisunsafe` 的指令，所以寫一篇筆記。

## SSL 憑證不安全的原因

當看到瀏覽器顯示 SSL 安全警告時，通常是因為以下原因：

* SSL 憑證是自簽名的，而非由受信任的憑證機構簽發。
* SSL 憑證已經過期。
* 憑證與網站的網域名稱不匹配。
* 你的瀏覽器不信任發出此憑證的機構。
* 內部開發環境使用的測試憑證。

這些警告是瀏覽器的安全機制，提醒當前連線可能不安全。

## Chrome 繞過 SSL 警告的方法

### 標準情況

正常如果遇到瀏覽器認為 SSL 不安全時，可以按進階，繼續瀏覽，如下圖：

![SSL 警告進階選項畫面](images/%E7%B9%BC%E7%BA%8C%E7%80%8F%E8%A6%BD%E4%BD%BF%E7%94%A8%E4%B8%8D%E5%AE%89%E5%85%A8%20SSL%20%E6%86%91%E8%AD%89%E7%B6%B2%E7%AB%99%E7%9A%84%E6%96%B9%E6%B3%95/ssl-warning-advanced-option.png)

步驟：

* 點擊「進階」選項。
* 點擊「繼續前往網站(不安全)」的連結。
* 瀏覽器會帶你進入目標網站。

### 特殊情況（沒有繼續瀏覽選項）

但有時會遇到無繼續瀏覽的情況，如下圖：

![SSL 警告無繼續瀏覽選項](images/%E7%B9%BC%E7%BA%8C%E7%80%8F%E8%A6%BD%E4%BD%BF%E7%94%A8%E4%B8%8D%E5%AE%89%E5%85%A8%20SSL%20%E6%86%91%E8%AD%89%E7%B6%B2%E7%AB%99%E7%9A%84%E6%96%B9%E6%B3%95/ssl-warning-no-proceed.png)

這個時候，可以滑鼠點 Chrome 瀏覽器，讓視窗 focus 在網頁上，然後：

* 鍵盤上直接輸入 `thisisunsafe`（不需要在任何輸入框中輸入）。
* 不會看到任何輸入痕跡，但輸入完成後，頁面會自動跳轉到實際網站。

注意：這個方法是一個隱藏功能，僅建議在你確定該網站安全的情況下使用，例如內部開發環境或測試環境。
